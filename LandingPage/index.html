Previous Health Records

Once a user logs in and uses the application to predict heart disease, a record is created and saved on the database, which the user can access at any time.

  • Saves current report on the database
  • Access to Previous record
  • Secured data logs

Vast and Reliable Dataset

This database contains 76 attributes, but all published experiments refer to using a subset of 14 of them. In particular, the Cleveland database is the only one that has been used by ML researchers to this date.

  • Dataset available at UCI
  • Contains 76 attributes
  • Report on subset of 14 such attributes

Analysis Using Python, Jupyter Notebook and Google Colab

The code used for modelling and analysing data for prediction the heart disease is pretty simple. The code is written using Jupyter Notebook and can also be easily accessed using google colab and python.

  • Available at GitHub
  • Easy to understand and edit
  • Written in Python, Google Colab and Jupyter Notebook

Future Scope

The application will be available on mobile devices in the future. Furthermore, this application can be used in combination with the latest health tracking devices to enable improved heart condition monitoring, which is now limited to simply displaying heart rate. This software can also be used in hospitals, where doctors or healthcare staff can look up a patient's past cardiac reports and, using the model, treat the problem ahead of time.
